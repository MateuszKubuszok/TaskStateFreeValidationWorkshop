Validation

Validation

Why

Existing ways of error handling

  • error flags/values
  • exceptions
  • Either
  • exclusive or: Xor, disjunction \/
Mention that from Scala 2.12 Xor is a type alias for Either - before it was a separate implementation because Either was not biased.

3 flavors of polymorphism

  • subtyping: most often as interface inheritance
  • parametric polymorphism: generics, templates, parametrized classes/methods/functions
  • ad hoc polymorphism: function/operator overload

Type class

  • based on ad hoc polymorphism
  • puts constraint on a type, by requiring overloading of some functions
  • is not a class in OOP sense (!)

Metaphor of assembly line

  • having boxes of some sort
  • doing something with things in boxes, one thing at a time

Functors

  • map takes a functor and function transforming its contents, and apply it to all elements contained in functor
trait MyFunctor[T] {
  def map[U](func: T => U): MyFunctor[U]
}
val box: Seq[Int] = Seq(1, 2, 3)
val func: Int => String = _.toString
val box2: Seq[String] = box.map(func) // Seq("1", "2", "3")

Functor laws

  • identity should leave a functor (type) intact: 
    def identity[T](value: T): T = value
val box3: Box[Int] = box.map(identity) // box3 == box
  • applying functions one by one, or composing them and then applying should not change the result
    val box = Seq(1, 2, 3)
val f1: Int => Int = _ * 2
val f2: Int => Double: _.toDouble
box.map(f1).map(f2) == box.map(a.andThen(b))
// both should give Seq(2.0, 4.0, 6.0)

Applicative functors

  • another type class
  • defines pure function for wrapping values
  • extends functor by handling functions stored within as a value
trait MyApplicative[T] extends MyFunctor[T] {
  def ap[U](other: MyApplicative[T => U]): MyApplicative[U]
}
val apv: MyApplicative[Int] = MyApplicative(1)
val apf: MyApplicative[String => Int] =
  MyApplicative((s: String) => s.toInt)
val apr: MyApplicative[Int] = apv.ap(apf)
Mention that constructor should be names pure or point.

Applicative laws

  • identity
    applicative.ap(MyApplicative(identity))
  • homomorphism
    MyApplicative(value).ap(MyApplicative(func))
  == MyApplicative(func(value))
  • interchange
    MyApplicative(value).ap(func)
  == MyApplicative(f => f(value)).ap(func)
  • ap is associative - composing like plain functions

Comparison

// MyFunctor[T]
def map[U](func: T => U): MyFunctor[U]

// MyApplicative[T]
def ap[U](other: MyApplicative[T => U]): MyApplicative[U]

Wrapping values - cats

sealed abstract class Validated[+E, +S]
object Validated {
  case class Valid[+S](a: S) extends Validated[Nothing, S]
  case class Invalid[+E](e: E) extends Validated[E, Nothing]

  def invalid[E, S](e: E): Validated[E, S] = Invalid(e)
  def valid[E, S](s: B): Validated[E, S] = Valid(s)
}
val s: Validated[String, Int] = Validated.valid(12)
val f: Validated[String, Int] = Validated.invalid("failed")

Wrapping values - Scalaz

sealed abstract class Validation[+E, +S]
case class Success[S](a: S) extends Validation[Nothing, S]
case class Failure[E](e: E) extends Validation[E, Nothing]

implicit class ValidationOps[V](val self: V) {
  def success[E]: Validation[E, V] = Success(self)
  def failure[S]: Validation[V, S] = Failure(self)
}
val s: Validation[String, Int] = 12.success[String]
val f: Validation[String, Int] = "failed".failure[Int]

Non empty list

NEL in Validated

type ValidatedNel[E, S] = Validated[NonEmptyList[E], S]

sealed abstract class Validated[+E, +S] {
  def toValidatedNel[EE >: E, SS >: S]: ValidatedNel[EE, SS] =
    this match {
      case s @ Valid(_) => s
      case Invalid(e)   => Validated.invalidNel(e)
    }
}

object Validated {
  def invalidNel[E, S](e: E): Validated[E, S] =
    Invalid(NonEmptyList(e))
}

NEL in Validated

type ValidationNel[E, +S] = Validation[NonEmptyList[E], S]

sealed abstract class Validation[+E, +S] {
  def toValidationNel[EE >: E, SS >: S]: ValidationNel[EE, SS] =
    this match {
      case s @ Success(_) => s
      case Failure(e) => Failure(NonEmptyList(e))
    }
}

implicit class ValidationOps[V](val self: V) {
  def failureNel[S]: Validation[V, S] =
    Failure(NonEmptyList(self))
}

Task

Task

Why

  • Scala's Future is not a monad
  • Scalaz's Future interface is impractical
  • useful functions for sync and async execution
  • trampoline let us avoid stack overflow
Mention that, while there are some important implementation concerns, but overall main advantage is the API.

Metaphor of assembly line

  • having boxes of some sort
  • putting things into boxes
  • taking things out of the box by one, doing something with them and packing back into boxes of the same kind

Monad type class

  • type constructor
  • unit function (constructor) (putting things into the box for the first time)
    def unit[T]: T => Monad[T]
  • flatMap function (taking something out of the box and deciding what to do with it)
    trait Monad[T] {
  def flatMap[U](func: T => Monad[U]) => Monad[U]
}
Mention the map -> flatten thing as well as where it doesn't apply (Future).

Monadic laws

  • left identity - aka left unit law
    unit(value).flatMap(func) == func(value)
  • right identity - aka left unit law
    mValue.flatMap(unit) == mValue
  • associativity - how we put "parenthesis" does not matter
    m.flatMap(f1).flatMap(f2) == m.flatMap(v=> f1(v).flatMap(f2))
Explain that Future (and Try) fails to fulfill left identity law, so it is not a monad, though it is enough to make for comprehension work.

Comparison

// MyFunctor[T]
def map[U](func: T => U): MyFunctor[U]

// MyApplicative[T]
def ap[U](other: MyApplicative[T => U]): MyApplicative[U]

// MyMonad[T]
def flatMap[U](func: T => MyMonad[U]): MyMonad[U]

State

case class State[S, A](val runF: S => (S, A)]) {

  def run(init: S): (S, A) = runF(init)

  def flatMap[B](fas: A => State[S, B]): State[S, B] =
    State[S,B](s => runF(s) match { case (s2, a) =>
      fas(a).run(s2)
    })
}
Mention that it is a little bit more complex in both Cats and Scalaz.

How - interpreter pattern

  • imagine your high level constructs as elements of some programming language
  • turn it into AST which elements will express param and returned value types
  • implement function that would take that virtual AST and run calculations for each element
  • you effectively implemented interpreter working directly on your AST

How - natural transformation

type ~>[-F[_], +G[_]] = NaturalTransformation[G, H]

trait NaturalTransformation[-F[_], +G[_]] { self =>

  def apply[A](fa: F[A]): G[A]

  def compose[E[_]](f: E ~> F): E ~> G = new (E ~> G) {
    def apply[A](ea: E[A]) = self(f(ea))
  }
  def andThen[H[_]](f: G ~> H): F ~> H = f compose self
}

def seqToSet: = new (Seq ~> Set) {
  override apply[T](seq: Seq[T]): Set[T] = seq.toSet
}
Explain how natural transformation works.

How - natural transformation

sealed trait UserDSL[Result]
case object GetUser extends UserDSL[User]
case class UpdateUser(user: User) extends UserDSL[User]
case class GetPreferences(user: User)
  extends UserDSL[Set[Preferences]]

object FutureInterpreter extends (UserDSL ~> Future) {
  def apply[Result](command: UserDSL[Result]): Future[Result] =
    command match {
      case GetUser => Future { /* get user */ }
      case UpdateUser(user) => Future { /* update user */ }
      case GetPreferences(user) => Future { /* get prefs */ }
    }
}

How - elements of Free

We have 3 cases to consider

  1. storing Pure value (aka Return) - we can extract it
  2. Suspending calculation - this we can interpret!
  3. storing FlatMapped calculation (aka Gosub)
sealed abstract class Free[S[_], A]
case class Pure[S[_], A](a: A) extends Free[S, A]
case class Suspend[S[_], A](a: S[A]) extends Free[S, A]
case class FlatMapped[S[_], A0, B](
    a0: Free[S, A0],
    f0: A0 => Free[S, B]
) extends Free[S, A]
Pure -> store value, Suspend -> store AST, FlatMapped -> x flatMap y = FlatMapped(x, y).

How - lifting

  • take function F -> G
  • put some H in between: F -> H -> G
  • we can say that we lifted F to H

How - lifting to Free

We lift our DSL to Free, it can use natural transformation to get us to the actual computation.

class Ops {
  def getUser: Free[UserDSL, User]
    = Free.liftF(GetUser)
  def updateUser(user: User): Free[UserDSL, User]
    = Free.liftF(UpdateUser(user))
  def getPreferences(user: User): Free[UserDSL, Set[Preference]]
    = Free.liftF(GetPreferences(user))
}
Lifting uses suspend. Why we use Ops instead of object or plain values I'll explain later.

Finally

val ops = new Ops
val interpreter: (UserDSL ~> Future)
val getUserWithPrefs: Free[UserDSL, (User, Set[Preference])] =
  for {
    user  <- ops.getUser
    prefs <- ops.getUserPreferences(user)
  } yield (user, prefs)

val future: Future[(User, Set[Preference])] =
    getUserWithPrefs.foldMap(interpreter)
Lifting uses suspend.

object ScalazCoproductUtils {
  sealed abstract class :+:[F[_], G[_]] {
    type 位[A] = Coproduct[F, G, A]
  }

  implicit class ComposeInterpreters[F[_], H[_]](f: F ~> H) {
    def :+:[G[_]](g: G ~> H): (G :+: F)#位 ~> H =
      new ((G :+: F)#位 ~> H) {
        def apply[A](fa: (G :+: F)#位[A]) = fa.run.fold(g, f)
      }
  }
}

type ConfigState[A] = State[Config, A]
final class CommandIdInterpreter(...)
    extends (Command ~> ConfigState) {

  // no state is actually stored within interpreter right now!
  override def apply[A](fa: Command[A]): ConfigState[A] =
    fa match {
      case GetNextAction(question) =>
        State.state[Config, ActionType](getNextAction(question))
      case Configure(question) =>
        State.put(configure(question))
      case Quit =>
        State.state[Config, Unit](quit())
    }
